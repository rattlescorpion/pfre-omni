# Testing & QA Prompt

## Purpose
Guidance for building reliable tests with PHPUnit and Pest. Use this when writing unit tests, feature tests, browser tests, or establishing test coverage standards.

## Testing Framework Setup

### Current Stack
- **Unit/Feature Tests**: Pest (Laravel's modern testing framework)
- **Browser Tests**: Laravel Dusk (if needed for E2E)
- **Configuration**: [phpunit.xml](phpunit.xml)
- **Test Directory**: [tests/](tests/)

### Installation & Setup
```bash
# Pest is pre-installed via composer
# Verify:
composer show | grep pestphp

# Run all tests:
composer test

# Run specific test:
php artisan test tests/Feature/LeadApiTest.php
```

## Test Lifecycle

### Setup & Teardown
```php
uses(Tests\TestCase::class);

beforeEach(function () {
    // Runs before each test
    $this->user = User::factory()->create();
    $this->actingAs($this->user);
});

afterEach(function () {
    // Runs after each test
    // Cleanup happens automatically with database transactions
});
```

### Database Transactions
```php
// Tests automatically run in transactions and rollback
// Database state is clean between tests

public function test_creates_lead()
{
    Lead::create(['name' => 'John', 'email' => 'john@test.com']);
    assertDatabaseHas('leads', ['email' => 'john@test.com']);
}
// Database automatically rolls back after test
```

## Writing Tests with Pest

### 1. Basic Unit Test
```php
// tests/Unit/Models/LeadTest.php

uses(Tests\TestCase::class);

test('lead has valid attributes', function () {
    $lead = Lead::factory()->create([
        'name' => 'John Doe',
        'email' => 'john@test.com'
    ]);
    
    expect($lead->name)->toBe('John Doe');
    expect($lead->email)->toBe('john@test.com');
    expect($lead->status)->toBe('new');  // Default
});

test('lead can be marked as contacted', function () {
    $lead = Lead::factory()->create();
    
    $lead->markAsContacted();
    
    expect($lead->status)->toBe('contacted');
});
```

### 2. Feature Test - List Endpoint
```php
// tests/Feature/Api/LeadApiTest.php

uses(Tests\TestCase::class)->beforeEach(function () {
    $this->user = User::factory()->create();
});

test('user can list leads', function () {
    Lead::factory(10)->create();
    
    $response = $this->actingAs($this->user)->getJson('/api/leads');
    
    $response->assertOk()
        ->assertJsonStructure([
            'data' => [['id', 'name', 'email', 'status']],
            'meta' => ['total', 'per_page', 'current_page']
        ])
        ->assertJsonCount(10, 'data');
});

test('list leads returns 401 without auth', function () {
    $response = $this->getJson('/api/leads');
    
    $response->assertUnauthorized();
});

test('user can filter leads by status', function () {
    Lead::factory(5)->create(['status' => 'active']);
    Lead::factory(3)->create(['status' => 'inactive']);
    
    $response = $this->actingAs($this->user)
        ->getJson('/api/leads?status=active');
    
    $response->assertOk()
        ->assertJsonCount(5, 'data');
});

test('user can search leads', function () {
    Lead::factory()->create(['name' => 'John Doe', 'email' => 'john@test.com']);
    Lead::factory()->create(['name' => 'Jane Smith', 'email' => 'jane@test.com']);
    
    $response = $this->actingAs($this->user)
        ->getJson('/api/leads?search=john');
    
    $response->assertOk()
        ->assertJsonCount(1, 'data')
        ->assertJsonPath('data.0.name', 'John Doe');
});
```

### 3. Feature Test - Create Endpoint
```php
test('user can create lead', function () {
    $data = [
        'name' => 'John Doe',
        'email' => 'john@test.com',
        'phone' => '+919876543210',
        'status' => 'new',
        'source' => 'website'
    ];
    
    $response = $this->actingAs($this->user)
        ->postJson('/api/leads', $data);
    
    $response->assertCreated()
        ->assertJsonPath('data.name', 'John Doe')
        ->assertJsonPath('data.email', 'john@test.com');
    
    assertDatabaseHas('leads', $data);
});

test('validation fails without required fields', function () {
    $response = $this->actingAs($this->user)->postJson('/api/leads', [
        'name' => '',  // Missing required field
        'email' => 'invalid-email'  // Invalid email
    ]);
    
    $response->assertUnprocessable()
        ->assertJsonValidationErrors(['name', 'email']);
});

test('email must be unique', function () {
    $existing = Lead::factory()->create(['email' => 'john@test.com']);
    
    $response = $this->actingAs($this->user)->postJson('/api/leads', [
        'name' => 'Jane',
        'email' => 'john@test.com'  // Duplicate
    ]);
    
    $response->assertUnprocessable()
        ->assertJsonValidationErrors('email');
});
```

### 4. Feature Test - Update Endpoint
```php
test('user can update own lead', function () {
    $lead = Lead::factory()->create(['owner_id' => $this->user->id]);
    
    $response = $this->actingAs($this->user)
        ->patchJson("/api/leads/{$lead->id}", [
            'status' => 'contacted'
        ]);
    
    $response->assertOk()
        ->assertJsonPath('data.status', 'contacted');
    
    assertDatabaseHas('leads', [
        'id' => $lead->id,
        'status' => 'contacted'
    ]);
});

test('user cannot update other users lead', function () {
    $other = User::factory()->create();
    $lead = Lead::factory()->create(['owner_id' => $other->id]);
    
    $response = $this->actingAs($this->user)
        ->patchJson("/api/leads/{$lead->id}", ['status' => 'contacted']);
    
    $response->assertForbidden();
});
```

### 5. Feature Test - Delete Endpoint
```php
test('user can delete own lead', function () {
    $lead = Lead::factory()->create(['owner_id' => $this->user->id]);
    
    $response = $this->actingAs($this->user)
        ->deleteJson("/api/leads/{$lead->id}");
    
    $response->assertNoContent();
    assertModelMissing($lead);
});

test('user cannot delete other users lead', function () {
    $other = User::factory()->create();
    $lead = Lead::factory()->create(['owner_id' => $other->id]);
    
    $response = $this->actingAs($this->user)
        ->deleteJson("/api/leads/{$lead->id}");
    
    $response->assertForbidden();
    assertModelExists($lead);
});
```

## Factories (Test Data)

### Create Factory
```php
// database/factories/LeadFactory.php

namespace Database\Factories;

use Illuminate\Database\Eloquent\Factories\Factory;

class LeadFactory extends Factory
{
    public function definition(): array
    {
        return [
            'name' => fake()->name(),
            'email' => fake()->unique()->safeEmail(),
            'phone' => fake()->phoneNumber(),
            'status' => 'new',
            'source' => fake()->randomElement(['website', 'referral', 'cold_call']),
            'owner_id' => User::factory(),
        ];
    }
    
    // State for specific scenarios
    public function active(): static
    {
        return $this->state([
            'status' => 'active'
        ]);
    }
    
    public function contacted(): static
    {
        return $this->state([
            'status' => 'contacted'
        ]);
    }
}
```

### Use Factory in Tests
```php
// Create single instance
$lead = Lead::factory()->create();

// Create multiple
$leads = Lead::factory(10)->create();

// Create with attributes
$lead = Lead::factory()->create([
    'name' => 'John Doe',
    'email' => 'john@test.com'
]);

// Use state
$lead = Lead::factory()->active()->create();

// Create instance without saving
$lead = Lead::factory()->make();
```

## Test Patterns & Anti-Patterns

### ✅ Good Patterns
```php
// ✅ Clear test name
test('authenticated user can create lead', function () {
    // ...
});

// ✅ Arrange-Act-Assert pattern
test('lead status updates correctly', function () {
    // Arrange
    $lead = Lead::factory()->create(['status' => 'new']);
    
    // Act
    $lead->update(['status' => 'contacted']);
    
    // Assert
    expect($lead->status)->toBe('contacted');
});

// ✅ Test one thing per test
test('email validation works', function () {
    $validator = Validator::make(['email' => 'invalid'], ['email' => 'email']);
    expect($validator->fails())->toBeTrue();
});

// ✅ Use descriptive assertions
expect($response->status())->toBe(201);  // Clear
// Not: expect($response)->toBeTruthy();  // Vague
```

### ❌ Anti-Patterns
```php
// ❌ Unclear test name
test('it works', function () {
    // ...
});

// ❌ Testing multiple things
test('lead crud operations', function () {
    // Create, read, update, delete all in one test
});

// ❌ Dependent tests
test('first test') { /* ... */ }
test('second test assumes first ran') { /* ... */ }

// ❌ Hard-coded values
test('calculates total', function () {
    expect(1 + 1)->toBe(2);  // This is always true, not testing logic
});
```

## Mocking & Stubbing

### Mock External Services
```php
test('sends notification via external service', function () {
    Http::fake([
        'notification-api.com/*' => Http::response(['status' => 'sent'], 200)
    ]);
    
    $service = new NotificationService();
    $result = $service->send('message');
    
    expect($result)->toBeTrue();
    Http::assertSent(function ($request) {
        return $request->url() === 'notification-api.com/send';
    });
});

// Stub failed response
Http::fake([
    '*' => Http::response(status: 500)
]);
```

### Spy on Methods
```php
test('calls email service', function () {
    $spy = Spy::on(EmailService::class, 'send');
    
    $service = new NotificationService();
    $service->notifyUser($user);
    
    expect($spy)->toHaveBeenCalledWith($user->email);
});
```

## Test Organization Best Practices

### Group Related Tests
```php
describe('Lead API', function () {
    describe('list endpoint', function () {
        test('returns paginated leads', function () { /* ... */ });
        test('filters by status', function () { /* ... */ });

    });
    
    describe('create endpoint', function () {
        test('creates lead with valid data', function () { /* ... */ });
        test('validates required fields', function () { /* ... */ });
    });

```

uses(Tests\TestCase::class)
