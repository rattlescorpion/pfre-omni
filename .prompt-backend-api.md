# Backend API Design Prompt

## Purpose
Guide for building RESTful APIs aligned with PFRE-Omni's architecture. Use this when creating new endpoints, designing request/response formats, implementing authentication, or reviewing API consistency.

## API Architecture Overview

### Base Structure
- **Base URL**: `/api/v1/` (versioning ready)
- **Format**: JSON exclusively
- **Authentication**: Laravel Sanctum (Bearer tokens)
- **Authorization**: Role-based access control (RBAC) via policies
- **Rate Limiting**: Configured in [routes/rate-limiting.php](routes/rate-limiting.php)
- **CORS**: Configured in [config/cors.php](config/cors.php)

### Location
- **Routes**: [routes/api.php](routes/api.php)
- **Controllers**: [app/Http/Controllers/](app/Http/Controllers/)
- **Middleware**: [app/Http/Middleware/](app/Http/Middleware/)
- **Requests**: [app/Http/Requests/](app/Http/Requests/) (validation)
- **Resources**: [app/Http/Resources/](app/Http/Resources/) (response transformation)

## HTTP Verbs & Status Codes

### Standard Verbs
```
GET     /api/leads              → List leads (paginated)
POST    /api/leads              → Create new lead
GET     /api/leads/{id}         → Get single lead
PUT     /api/leads/{id}         → Replace entire lead
PATCH   /api/leads/{id}         → Partial update
DELETE  /api/leads/{id}         → Delete lead
```

### Status Codes
```
200 OK                  ← Success (GET, PUT, PATCH)
201 Created             ← Resource created (POST)
202 Accepted            ← Async job queued (background processing)
204 No Content          ← Success with no response body (DELETE)
400 Bad Request         ← Client error (invalid params)
401 Unauthorized        ← Missing or invalid authentication
403 Forbidden           ← Authenticated but lacks permission
404 Not Found           ← Resource doesn't exist
409 Conflict            ← Request conflicts with current state
422 Unprocessable       ← Validation error (details in response)
429 Too Many Requests   ← Rate limited
500 Internal Server     ← Server error
503 Unavailable         ← Service temporarily unavailable
```

## Creating RESTful Endpoints

### 1. List Endpoint with Filtering & Pagination
```php
// routes/api.php
Route::get('/leads', [LeadController::class, 'index'])->name('leads.index');

// app/Http/Controllers/LeadController.php
class LeadController extends Controller
{
    public function index(Request $request)
    {
        $query = Lead::query();
        
        // Filter by status
        if ($request->has('status')) {
            $query->where('status', $request->input('status'));
        }
        
        // Search by name or email
        if ($request->has('search')) {
            $search = $request->input('search');
            $query->where(function($q) use ($search) {
                $q->where('name', 'like', "%{$search}%")
                  ->orWhere('email', 'like', "%{$search}%");
            });
        }
        
        // Eager load relationships (prevent N+1)
        $query->with('owner', 'property', 'followUps');
        
        // Sort
        $sortBy = $request->input('sort_by', 'created_at');
        $sortOrder = $request->input('sort_order', 'desc');
        $query->orderBy($sortBy, $sortOrder);
        
        // Paginate
        $leads = $query->paginate($request->input('per_page', 15));
        
        return LeadResource::collection($leads);
    }
}
```

**Expected Request**:
```
GET /api/leads?status=active&search=john&sort_by=name&sort_order=asc&per_page=20
```

### 2. Show Endpoint
```php
Route::get('/leads/{id}', [LeadController::class, 'show'])->name('leads.show');

class LeadController extends Controller
{
    public function show(Lead $lead)
    {
        // Route model binding + authorization
        $this->authorize('view', $lead);
        
        return new LeadResource($lead->load('owner', 'property', 'followUps'));
    }
}
```

### 3. Create Endpoint with Validation
```php
Route::post('/leads', [LeadController::class, 'store'])->name('leads.store');

class LeadController extends Controller
{
    public function store(StoreLead $request)
    {
        $lead = Lead::create($request->validated());
        
        // Dispatch event for notifications, logging
        LeadCreated::dispatch($lead);
        
        return response()->json([
            'data' => new LeadResource($lead),
            'message' => 'Lead created successfully'
        ], 201);
    }
}

// app/Http/Requests/StoreLead.php
class StoreLead extends FormRequest
{
    public function authorize(): bool
    {
        return $this->user()->can('create', Lead::class);
    }
    
    public function rules(): array
    {
        return [
            'name' => 'required|string|max:255',
            'email' => 'required|email|unique:leads,email',
            'phone' => 'nullable|phone:IN',  // Indian phone format
            'status' => 'in:new,contacted,qualified,proposal,closed',
            'property_id' => 'nullable|exists:properties,id',
            'source' => 'in:website,referral,cold_call,email',
        ];
    }
}
```

### 4. Update Endpoint
```php
Route::patch('/leads/{id}', [LeadController::class, 'update'])->name('leads.update');

class LeadController extends Controller
{
    public function update(UpdateLead $request, Lead $lead)
    {
        $this->authorize('update', $lead);
        
        $lead->update($request->validated());
        
        LeadUpdated::dispatch($lead);
        
        return new LeadResource($lead);
    }
}

// app/Http/Requests/UpdateLead.php
class UpdateLead extends FormRequest
{
    public function authorize(): bool
    {
        return $this->user()->can('update', $this->lead());
    }
    
    public function rules(): array
    {
        return [
            'name' => 'sometimes|string|max:255',
            'email' => 'sometimes|email|unique:leads,email,' . $this->lead()->id,
            'phone' => 'nullable|phone:IN',
            'status' => 'in:new,contacted,qualified,proposal,closed',
        ];
    }
}
```

### 5. Delete Endpoint
```php
Route::delete('/leads/{id}', [LeadController::class, 'destroy'])->name('leads.destroy');

class LeadController extends Controller
{
    public function destroy(Lead $lead)
    {
        $this->authorize('delete', $lead);

}
```

## Resource (Response Transformation)

### Define LeadResource
```php
// app/Http/Resources/LeadResource.php
class LeadResource extends JsonResource
{
    public function toArray($request): array
    {
        return [
            'id' => $this->id,
            'name' => $this->name,
            'email' => $this->email,
            'phone' => $this->phone,
            'status' => $this->status,
            'source' => $this->source,
            'property' => new PropertyResource($this->whenLoaded('property')),
            'owner' => new UserResource($this->whenLoaded('owner')),
            'follow_ups' => FollowUpResource::collection($this->whenLoaded('followUps')),
            'created_at' => $this->created_at?->toIso8601String(),
            'updated_at' => $this->updated_at?->toIso8601String(),
        ];
    }
}
```

### Conditional Resource Loading
```php
'owner' => new UserResource($this->whenLoaded('owner')),
// Only includes owner if relationship was eager-loaded
```

## Authentication & Authorization

### 1. Bearer Token Authentication
```php
// Middleware automatically validates Sanctum token
Route::middleware('auth:sanctum')->group(function () {
    Route::get('/profile', [UserController::class, 'profile']);
    Route::post('/logout', [AuthController::class, 'logout']);
});

// In requests:
// Authorization: Bearer {token}
```

### 2. Role-Based Authorization (Policies)
```php
// app/Policies/LeadPolicy.php
class LeadPolicy
{
    public function viewAny(User $user): bool
    {
        return $user->hasRole('admin', 'manager', 'sales');
    }
    
    public function view(User $user, Lead $lead): bool
    {
        return $user->id === $lead->owner_id || $user->hasRole('admin');
    }
    
    public function create(User $user): bool
    {
        return $user->hasRole('admin', 'manager', 'sales');
    }
    
    public function update(User $user, Lead $lead): bool
    {
        return $user->id === $lead->owner_id || $user->hasRole('admin');
    }
    
    public function delete(User $user, Lead $lead): bool
    {
        return $user->hasRole('admin');
    }
}

// Use in controller:
$this->authorize('view', $lead);  // Throws 403 if unauthorized
```

## Input Validation Best Practices

### 1. Use Form Request Classes
```php
// ❌ Bad: Validation in controller
public function store(Request $request)
{
    $request->validate([...]);
}

// ✅ Good: Separate request class
public function store(StoreLead $request)
{
    // $request is already validated
    Lead::create($request->validated());
}
```

### 2. Custom Validation Rules
```php
// In StoreLead request
public function rules(): array
{
    return [
        'email' => [
            'required',
            'email',
            'unique:leads,email',
            function (string $attribute, mixed $value, Closure $fail) {
                if (blockedEmailDomains()->contains($value)) {
                    $fail('Email domain is blocked.');
                }
            }
        ]
    ];
}
```

### 3. Handle Validation Errors
```php
// Automatically returns 422 with validation errors
// In app/Http/Controllers/Controller.php or middleware
protected function failedValidation(Validator $validator)
{
    throw new HttpResponseException(response()->json([
        'message' => 'Validation failed',
        'errors' => $validator->errors(),
        'status' => 'error'
    ], 422));
}
```

## Query Optimization

### 1. Eager Load Relationships
```php
// ❌ Bad: N+1 query problem
$leads = Lead::all();
foreach ($leads as $lead) {
    echo $lead->owner->name;  // Query per lead
}

// ✅ Good: Eager load
$leads = Lead::with('owner')->get();
foreach ($leads as $lead) {
    echo $lead->owner->name;  // No extra queries
}
```

### 2. Lazy Eager Loading (for large datasets)
```php
$leads = Lead::all();
$leads->load('owner', 'property');  // Load after fetch
```

### 3. Selective Field Loading (sparse fieldsets)
```php
// Client requests only specific fields
GET /api/leads?fields=id,name,email

// In controller:
$fields = $request->input('fields', 'id,name,email');
$leads = Lead::select(explode(',', $fields))->get();
```

## Error Handling

### 1. Custom Exception Handler
```php
// app/Exceptions/Handler.php
public function render($request, Throwable $exception)
{
    if ($exception instanceof ModelNotFoundException) {
        return response()->json([
            'message' => 'Resource not found',
            'status' => 'error'
        ], 404);
    }
    
    if ($exception instanceof AuthorizationException) {
        return response()->json([
            'message' => 'You are not authorized to perform this action',
            'status' => 'error'
        ], 403);
    }
    
    if ($exception instanceof ValidationException) {
        return response()->json([
            'message' => 'Validation failed',
            'errors' => $exception->errors(),
            'status' => 'error'
        ], 422);
    }
    
    // Log other exceptions
    \Log::error($exception->getMessage());
    
    return response()->json([
        'message' => 'An unexpected error occurred',
        'status' => 'error'
    ], 500);
}
```

## Rate Limiting

### Configure in routes/rate-limiting.php
```php
// Limit API requests to 60 per minute per user
RateLimiter::for('api', function (Request $request) {
    return Limit::perMinute(60)->by(
        $request->user()?->id ?: $request->ip()
    );
});

// In routes/api.php:
Route::middleware('throttle:api')->group(function () {
    // API routes
});
```

### Handle Rate Limit Response
```json
{
  "message": "Too Many Requests",
  "status": "error"
}
```

Response headers include:
```
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 45
X-RateLimit-Reset: 1621234567
```

## Request Logging & Monitoring

### Log All API Requests
```php
// app/Http/Middleware/LogApiRequests.php
class LogApiRequests
{
    public function handle($request, Closure $next)
    {
        $start = microtime(true);
        $response = $next($request);
        $duration = microtime(true) - $start;
        
        \Log::info('API Request', [
            'method' => $request->method(),
            'path' => $request->path(),
            'status' => $response->status(),
            'duration_ms' => round($duration * 1000, 2),
            'user_id' => $request->user()?->id,
        ]);
        
        return $response;
    }
}

// Register in Http/Kernel.php
protected $middlewareGroups = [
    'api' => [
        LogApiRequests::class,
        // ...
    ]
];
```

## Common Patterns

### Bulk Operations
```php
// Bulk update
POST /api/leads/bulk-update
{
  "ids": [1, 2, 3],
  "data": { "status": "contacted" }
}

// Bulk delete
DELETE /api/leads/bulk-delete
{
  "ids": [1, 2, 3]
}
```

### Async Operations
```php
// Long-running operation returns 202 Accepted
POST /api/reports/generate
{
  "report_type": "sales_summary",
  "start_date": "2026-01-01",
  "end_date": "2026-05-02"
}

Response: 202 Accepted
{
  "job_id": "uuid-1234",
  "status_url": "/api/jobs/uuid-1234"
}
```

### Webhooks (Event Notifications)
```php
// Register webhook
POST /api/webhooks
{
  "event": "lead.created",
  "url": "https://client.com/webhook"
}

// Trigger on event
LeadCreated::dispatch($lead);

// POST to webhook URL:
{
  "event": "lead.created",
  "timestamp": "2026-05-02T10:30:00Z",
  "data": { "id": 1, "name": "John", ... }
}
```

## Testing API Endpoints

### Feature Test Example
```php
// tests/Feature/LeadApiTest.php
public function test_can_list_leads()
{
    $user = User::factory()->create();
    Lead::factory(10)->create();
    
    $response = $this->actingAs($user)->getJson('/api/leads');
    
    $response->assertStatus(200)
        ->assertJsonStructure([
            'data' => [['id', 'name', 'email', 'status']],
            'meta' => ['total', 'per_page', 'current_page']
        ]);
}

public function test_validation_error_returns_422()
{
    $user = User::factory()->create();
    
    $response = $this->actingAs($user)->postJson('/api/leads', [
        'name' => ''  // Missing required field
    ]);
    
    $response->assertStatus(422)
        ->assertJsonValidationErrors('name');
}
```
