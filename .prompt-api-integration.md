# Backend API Integration Prompt

## Purpose
Guide for building Axios-driven Blade views with seamless API communication, error handling, validation feedback, and loading states. Use this when creating forms, data fetches, or interactive components that communicate with the Laravel REST API.

## When to Use
- Building forms that submit to API endpoints
- Creating data-heavy pages that fetch via AJAX instead of server-side rendering
- Adding real-time validation or auto-save features
- Implementing multi-step workflows with API calls between steps
- Building dashboard components that refresh via API polling

## Tech Stack for API Integration
- **HTTP Client**: Axios (configured in [resources/js/app.js](resources/js/app.js))
- **State Management**: Alpine.js (x-data for local form state) or Vue 5 (for complex components)
- **Error Handling**: Validation feedback in UI, error toasts, retry logic
- **Authentication**: Laravel Sanctum (Bearer token in request headers)
- **CORS**: Configured in `config/cors.php`

## Project API Structure
- **Base URL**: `/api/` (relative path in same domain)
- **Authentication**: Bearer token passed in `Authorization` header via Axios interceptor
- **Response format**: JSON with `data`, `status`, `message` structure
- **Errors**: HTTP status codes with JSON error details
- **API Routes**: Defined in [routes/api.php](routes/api.php)

## Axios Configuration Reference
```javascript
// In resources/js/app.js - Axios is pre-configured with:
window.axios = axios.create({
  baseURL: '/api',
  headers: {
    'X-Requested-With': 'XMLHttpRequest',
    'Authorization': `Bearer ${token}`
  }
});
```

## Common API Integration Patterns

### 1. Form Submission with Error Handling
```blade
<div x-data="{ 
  form: { email: '', name: '' },
  errors: {},
  submitting: false,
  success: false
}" class="space-y-4">

  <div>
    <label class="block text-sm font-medium dark:text-gray-200">Email</label>
    <input 
      x-model="form.email" 
      type="email" 
      class="w-full px-3 py-2 border dark:bg-gray-700 dark:text-white"
      :class="{ 'border-red-500': errors.email }"
    />
    <template x-if="errors.email">
      <p class="text-red-500 text-sm mt-1" x-text="errors.email[0]"></p>
    </template>
  </div>

  <button 
    @click="submitForm()"
    :disabled="submitting"
    class="bg-blue-500 hover:bg-blue-600 disabled:opacity-50 text-white px-4 py-2 rounded"
    :class="{ 'opacity-50 cursor-not-allowed': submitting }"
  >
    <span x-show="!submitting">Submit</span>
    <span x-show="submitting">Submitting...</span>
  </button>

  <template x-if="success">
    <div class="bg-green-100 border border-green-400 text-green-700 px-4 py-3 rounded">
      ✓ Form submitted successfully!
    </div>
  </template>

  <script>
    document.addEventListener('alpine:init', () => {
      Alpine.data('formData', () => ({
        submitForm() {
          this.submitting = true;
          this.errors = {};
          axios.post('/leads', this.form)
            .then(response => {
              this.success = true;
              this.form = { email: '', name: '' };
              setTimeout(() => { this.success = false; }, 3000);
            })
            .catch(error => {
              if (error.response?.status === 422) {
                this.errors = error.response.data.errors;
              }
            })
            .finally(() => { this.submitting = false; });
        }
      }));
    });
  </script>
</div>
```

### 2. Data Table with Lazy Loading & Search
```blade
<div x-data="{ 
  items: [],
  search: '',
  page: 1,
  loading: false,
  total: 0
}" class="space-y-4">

  <input 
    x-model="search"
    @keyup.debounce="page = 1; loadData()"
    placeholder="Search leads..."
    class="w-full px-3 py-2 border dark:bg-gray-700"
  />

  <table class="w-full border-collapse">
    <thead>
      <tr class="bg-gray-100 dark:bg-gray-800">
        <th class="px-4 py-2 text-left">Name</th>
        <th class="px-4 py-2 text-left">Email</th>
        <th class="px-4 py-2 text-left">Status</th>
      </tr>
    </thead>
    <tbody>
      <template x-for="item in items" :key="item.id">
        <tr class="border-b dark:border-gray-700">
          <td class="px-4 py-2" x-text="item.name"></td>
          <td class="px-4 py-2" x-text="item.email"></td>
          <td class="px-4 py-2">
            <span class="px-3 py-1 rounded text-sm" 
              :class="item.status === 'active' ? 'bg-green-100 text-green-700' : 'bg-gray-100 text-gray-700'"
              x-text="item.status">
            </span>
          </td>
        </tr>
      </template>
    </tbody>
  </table>

  <script>
    Alpine.data('dataTable', () => ({
      loadData() {
        this.loading = true;
        axios.get('/leads', {
          params: { search: this.search, page: this.page }
        })
          .then(response => {
            this.items = response.data.data;
            this.total = response.data.total;
          })
          .catch(error => console.error('Error loading data:', error))
          .finally(() => { this.loading = false; });
      },
      init() {
        this.loadData();
      }
    }));
  </script>
</div>
```

### 3. Multi-Step Form with API Calls Between Steps
```blade
<div x-data="{ 
  step: 1,
  formData: {},
  validating: false,
  error: null
}">

  <!-- Step 1: Personal Info -->
  <template x-if="step === 1">
    <div class="space-y-4">
      <h2 class="text-lg font-bold dark:text-white">Step 1: Personal Information</h2>
      <input x-model="formData.name" placeholder="Full Name" class="w-full px-3 py-2 border dark:bg-gray-700"/>
      <input x-model="formData.email" placeholder="Email" class="w-full px-3 py-2 border dark:bg-gray-700"/>
      <button @click="validateStep1()" class="bg-blue-500 text-white px-4 py-2 rounded">
        <span x-show="!validating">Next Step</span>
        <span x-show="validating">Validating...</span>
      </button>
    </div>
  </template>

  <!-- Step 2: Property Details -->
  <template x-if="step === 2">
    <div class="space-y-4">
      <h2 class="text-lg font-bold dark:text-white">Step 2: Property</h2>
      <input x-model="formData.property" placeholder="Property Address" class="w-full px-3 py-2 border dark:bg-gray-700"/>
      <button @click="step = 1" class="bg-gray-400 text-white px-4 py-2 rounded">Back</button>
      <button @click="submitForm()" class="bg-blue-500 text-white px-4 py-2 rounded">Submit</button>
    </div>
  </template>

  <template x-if="error">
    <div class="bg-red-100 border border-red-400 text-red-700 px-4 py-3 rounded" x-text="error"></div>
  </template>

  <script>
    Alpine.data('multiStepForm', () => ({
      validateStep1() {
        this.validating = true;
        axios.post('/leads/validate-step1', { 
          name: this.formData.name, 
          email: this.formData.email 
        })
          .then(() => {
            this.step = 2;
            this.error = null;
          })
          .catch(error => {
            this.error = error.response?.data?.message || 'Validation failed';
          })
          .finally(() => { this.validating = false; });
      },
      submitForm() {
        axios.post('/leads', this.formData)
          .then(() => {
            alert('Lead created successfully!');
            this.formData = {};
            this.step = 1;
          })
          .catch(error => {
            this.error = error.response?.data?.message || 'Submission failed';
          });
      }
    }));
  </script>
</div>
```

### 4. Real-Time Auto-Save Draft
```blade
<textarea 
  x-data="{ 
    content: '',
    lastSaved: null,
    autoSave() {
      axios.patch('/leads/draft-1', { content: this.content })
        .then(() => {
          this.lastSaved = new Date().toLocaleTimeString();
        })
        .catch(error => console.error('Auto-save failed:', error));
    }
  }"
  x-model="content"
  @input.debounce="autoSave()"
  class="w-full px-3 py-2 border dark:bg-gray-700 dark:text-white"
  placeholder="Start typing... (auto-saves)"
></textarea>
<p class="text-gray-500 text-sm" x-text="lastSaved ? 'Last saved: ' + lastSaved : ''"></p>
```

## Error Handling Best Practices

### 1. Validation Errors (422 Unprocessable Entity)
```javascript
.catch(error => {
  if (error.response?.status === 422) {
    // Validation errors from Laravel
    this.errors = error.response.data.errors; // { field: [message] }
  }
});
```

### 2. Authentication Errors (401 Unauthorized)
```javascript
.catch(error => {
  if (error.response?.status === 401) {
    // Token expired or invalid
    window.location.href = '/login';
  }
});
```

### 3. Server Errors (5xx)
```javascript
.catch(error => {
  if (error.response?.status >= 500) {
    // Generic server error message
    this.error = 'Server error. Please try again later.';
  }
});
```

### 4. Network Errors (No response)
```javascript
.catch(error => {
  if (!error.response) {
    // Network failure
    this.error = 'Network error. Check your connection.';
  }
});
```

## Loading States & UX Patterns

### Loading Skeleton (while fetching)
```blade
<div x-show="loading" class="space-y-4">
  <div class="h-12 bg-gray-300 dark:bg-gray-600 rounded animate-pulse"></div>
  <div class="h-12 bg-gray-300 dark:bg-gray-600 rounded animate-pulse"></div>
</div>
<div x-show="!loading"><!-- actual content --></div>
```

### Debounced Search (avoid excessive API calls)
```blade
<input 
  x-model="search"
  @keyup.debounce.500="performSearch()"
  placeholder="Search..."
/>
```

### Optimistic Updates (update UI before API confirmation)
```javascript
const originalItem = { ...item };
item.status = 'active'; // Update UI immediately

axios.patch(`/leads/${item.id}`, { status: 'active' })
  .catch(error => {
    item.status = originalItem.status; // Revert if failed
  });
```

## Response Format Conventions

**Expected API Response Structure:**
```json
{
  "data": { /* resource data */ },
  "message": "Success message",
  "status": "success"
}
```

**Paginated Response:**
```json
{
  "data": [ /* items array */ ],
  "meta": {
    "total": 100,
    "per_page": 15,
    "current_page": 1,
    "total_pages": 7
  }
}
```

**Error Response:**
```json
{
  "message": "Validation failed",
  "errors": {
    "email": ["Email is required", "Email must be valid"]
  }
}
```

## Axios Interceptors (in app.js)
```javascript
// Request interceptor: Add auth token
axios.interceptors.request.use(config => {
  const token = document.querySelector('meta[name="csrf-token"]')?.content;
  if (token) {
    config.headers['Authorization'] = `Bearer ${token}`;
  }
  return config;
});

// Response interceptor: Handle common errors
axios.interceptors.response.use(
  response => response,
  error => {
    if (error.response?.status === 401) {
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);
```

## CSRF Protection
All forms must include CSRF token from Blade:
```blade
<input type="hidden" name="_token" value="{{ csrf_token() }}">
<!-- Or as Axios header (automatic if configured) -->
```

## Mobile Responsiveness with API
- Use `debounce` on search inputs to reduce API calls on mobile
- Show loading skeleton instead of blocking UI
- Handle slow network scenarios gracefully
- Provide offline feedback if network is unreliable

## Performance Optimization
- **Pagination**: Fetch data in chunks (limit 15-50 per page)
- **Lazy loading**: Load related data only when needed
- **Request debouncing**: Debounce search (500ms) to avoid excessive calls
- **Caching**: Use Alpine store or localStorage for frequently accessed data
- **Compression**: Enable gzip in Vite config for production builds
