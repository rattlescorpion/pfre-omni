# Component Library Catalog Prompt

## Purpose
Reference guide for reusable Blade components with usage patterns, props, and examples. Use this when discovering what components exist, understanding how to use them, or deciding whether to create a new component or extend an existing one.

## Component Inventory

### Layout Components

**File**: [resources/views/layouts/app.blade.php](resources/views/layouts/app.blade.php)
**Purpose**: Root application layout with sidebar, topnav, and dark mode support
**Props**:

**Usage**:
```blade
<x-app-layout title="Dashboard">
  <div class="container mx-auto">

  </div>
```

### Data Display Components

#### `<x-kpi-card>`
**File**: [resources/views/components/kpi-card.blade.php](resources/views/components/kpi-card.blade.php)
**Purpose**: Metric card for dashboard with icon, value, trend
**Props**:
```blade
@props([
    'title' => '',           // Card title/label
    'value' => '',           // Main metric value
    'icon' => 'chart-line',  // Icon name (e.g., users, trending-up)
    'color' => 'blue',       // Color variant: blue, green, red, orange
    'trend' => null,         // Trend value with % symbol
    'trendUp' => true,       // Trend direction (up/down)
])
```
**Usage**:
```blade
<x-kpi-card 
  title="Total Leads" 
  value="1,234" 
  icon="users" 
  color="blue"
  trend="+12%"
  :trendUp="true"
/>
```
**Dark Mode**: ✅ Supports dark mode with `dark:` utilities
**Example**:
```blade
<div class="grid grid-cols-1 md:grid-cols-4 gap-6">
  <x-kpi-card title="Total Leads" value="1,234" color="blue" />
  <x-kpi-card title="Won Deals" value="456" color="green" />
  <x-kpi-card title="Revenue" value="$2.5M" color="orange" />
  <x-kpi-card title="Conversion" value="24%" color="purple" />
</div>
```

#### `<x-data-table>`
**File**: [resources/views/components/data-table.blade.php](resources/views/components/data-table.blade.php)
**Purpose**: Searchable, sortable table with pagination
**Props**:
```blade
@props([
    'headers' => [],          // Array of column headers
    'items' => [],            // Array of data rows
    'searchable' => true,     // Enable search input
    'sortable' => true,       // Enable column sorting
    'paginate' => true,       // Enable pagination
    'actions' => null,        // Action buttons slot
])
```
**Usage**:
```blade
<x-data-table 
  :headers="['Name', 'Email', 'Status', 'Actions']"
  :items="$leads"
  searchable
>
  @slot('actions', $item)
    <a href="/leads/{{ $item->id }}/edit" class="text-blue-500">Edit</a>
    <a href="/leads/{{ $item->id }}/delete" class="text-red-500">Delete</a>
  @endslot
</x-data-table>
```
**Dark Mode**: ✅ Full dark mode support
**Features**: Alpine.js search, click-to-sort columns, row striping

#### `<x-alert>`
**File**: [resources/views/components/alert.blade.php](resources/views/components/alert.blade.php)
**Purpose**: Dismissible alert message with colored variants
**Props**:
```blade
@props([
    'type' => 'info',       // success, error, warning, info
    'message' => '',        // Alert text
    'dismissible' => true,  // Show close button
])
```
**Usage**:
```blade
<x-alert type="success" message="Lead created successfully!" />
<x-alert type="error" message="An error occurred. Please try again." dismissible />
```
**Available Types**:
- `success` (green) — Positive confirmations
- `error` (red) — Errors and failures
- `warning` (yellow) — Alerts and cautions
- `info` (blue) — Informational messages

## Component Lifecycle & Patterns

### Creating a New Component

1. **Determine Type**: Is it a layout, data display, form, or interactive component?
2. **Check Existing**: Search component library first to avoid duplication
3. **Follow Naming**: Use kebab-case (e.g., `my-component.blade.php`)
4. **Define Props**: Use `@props([])` for clear API
5. **Dark Mode**: Always include `dark:` utilities
6. **Test**: Verify in layout with sample data

### Example: Creating a new component
```blade
{{-- resources/views/components/property-card.blade.php --}}
@props([
    'property' => null,
    'featured' => false,
])

<div class="bg-white dark:bg-gray-800 rounded-lg overflow-hidden shadow-sm hover:shadow-md transition-shadow p-4">
  @if($featured)
    <div class="absolute top-2 right-2 bg-yellow-500 text-white px-2 py-1 rounded text-xs font-bold">Featured</div>
  @endif
  
  <h3 class="text-lg font-bold dark:text-white">{{ $property->title }}</h3>
  <p class="text-gray-600 dark:text-gray-400 text-sm">{{ $property->location }}</p>
  <div class="mt-4 flex justify-between items-center">
    <span class="text-2xl font-bold dark:text-white">₹{{ number_format($property->price) }}</span>
    <a href="/properties/{{ $property->id }}" class="text-blue-500 hover:text-blue-700">View Details</a>
  </div>
</div>
```

**Usage**:
```blade
<div class="grid grid-cols-1 md:grid-cols-3 gap-4">
  @foreach($properties as $property)
    <x-property-card :property="$property" :featured="$property->featured" />
  @endforeach
</div>
```

## Slot Usage Patterns

### Named Slots (content sections)
```blade
{{-- Component definition --}}
<x-card>
  <x-slot name="header">
    <h2>Title</h2>
  </x-slot>
  
  {{ $slot }}  <!-- Default slot -->
  
  <x-slot name="footer">
    <button>Action</button>
  </x-slot>
</x-card>
```

### Scoped Slots (pass component data to parent)
```blade
{{-- Component definition --}}
@forelse($items as $item)
  {{ $slot($item) }}
@empty
  No items found
@endforelse

{{-- Usage --}}
<x-item-list :items="$items">
  @slot($item)
    <div>{{ $item->name }} - {{ $item->value }}</div>
  @endslot
</x-item-list>
```

## Performance Optimization for Components

### Avoid N+1 Queries
```blade
{{-- ❌ Bad: Queries inside loop --}}
@foreach($users as $user)
  <div>{{ $user->profile->name }}</div>
@endforeach

{{-- ✅ Good: Eager load relationship --}}
@foreach($users->load('profile') as $user)
  <div>{{ $user->profile->name }}</div>
@endforeach
```

### Cache Component Renders
```blade
{{-- Use Blade's @cache directive --}}
@cache
  <x-expensive-component :data="$data" />
@endcache(3600)
```

### Lazy Load Heavy Components
```blade
{{-- For dashboard widgets that can load after initial render --}}
<livewire:widget.analytics lazy />
```

## Component Testing

### Unit Test Pattern
```php
public function test_kpi_card_renders_with_props()
{
    $view = view('components.kpi-card', [
        'title' => 'Total Leads',
        'value' => '1,234',
        'color' => 'blue'
    ]);
    
    $this->assertStringContainsString('Total Leads', $view);
    $this->assertStringContainsString('1,234', $view);
}
```

### Browser Test Pattern (Pest)
```php
test('data table search works', function () {
    $this->visit('/leads')
        ->type('john', '[data-test=search-leads]')
        ->assertSee('John Doe');
});
```
