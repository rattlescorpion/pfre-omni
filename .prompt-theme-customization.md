## Common Customization Scenarios

### 1. Update Brand Colors (e.g., for White-Label Client)
**Scenario**: Create a new color scheme for a specific real-estate vertical or client brand.

**Steps**:
1. Add new CSS variables to [design-tokens.css](resources/css/design-tokens.css):
```css
/* Client: Luxury Estates */
--color-primary-500: #8b5cf6 (purple)
--color-primary-600: #7c3aed
--color-accent-500: #ec4899 (pink)
```

2. Create a variant Blade layout or CSS class:
```blade
<!-- In resources/views/layouts/app.blade.php -->
<html x-data="{ theme: 'default' }" :class="theme === 'luxury' ? 'theme-luxury' : ''">
```

3. Add CSS override:
```css
/* resources/css/app.css */
html.theme-luxury {
  --color-primary-500: #8b5cf6;
  --color-primary-600: #7c3aed;
  --color-accent-500: #ec4899;
}
```

### 2. Add Dark Mode Extended Palette
**Scenario**: Enhance dark mode with specific dark theme colors.

**Steps**:
```css
/* In design-tokens.css */
:root {
  --color-bg-light: #ffffff;
  --color-bg-dark: #111827;
  --color-text-light: #111827;
  --color-text-dark: #f9fafb;
}

html.dark {
  --color-bg: var(--color-bg-dark);
  --color-text: var(--color-text-dark);
}

html:not(.dark) {
  --color-bg: var(--color-bg-light);
  --color-text: var(--color-text-light);
}
```

**Usage in components**:
```blade
<div class="bg-[var(--color-bg)] text-[var(--color-text)]">
  Content that respects light/dark theme
</div>
```

### 3. Add New Semantic Color (e.g., Special Offer)
**Scenario**: Introduce a new semantic color for promotional or special deal features.

**Steps**:
```css
/* In design-tokens.css */
--color-special: #a78bfa (light purple)
--color-special-light: #ede9fe
--color-special-dark: #6d28d9

--color-premium: #fbbf24 (gold)
--color-premium-light: #fffbeb
--color-premium-dark: #b45309
```

**Usage in components**:
```blade
<div class="bg-[var(--color-special-light)] border border-[var(--color-special)] text-[var(--color-special)]">
  Special Offer Badge
</div>
```

### 4. Adjust Spacing Scale for Compact/Spacious Layouts
**Scenario**: Create a compact version for mobile or a spacious version for desktop.

**Steps**:
```css
/* Compact variant */
html.compact {
  --space-1: 0.125rem;
  --space-2: 0.25rem;
  --space-4: 0.5rem;
}

/* Spacious variant */
html.spacious {
  --space-4: 1.5rem;
  --space-6: 2rem;
  --space-8: 3rem;
}
```

**Usage**:
```blade
<div class="p-[var(--space-4)] space-y-[var(--space-2)]">
  <!-- spacing adapts based on html class -->
</div>
```

### 5. Create a High-Contrast Mode for Accessibility
**Scenario**: Add a high-contrast theme for accessibility compliance.

**Steps**:
```css
html.high-contrast {
  --color-text-light: #000000;
  --color-text-dark: #ffffff;
  --color-primary-500: #0000ff;
  --color-accent-500: #ff6600;
  --color-success: #008000;
  --color-danger: #ff0000;
}
```

**Toggle in UI**:
```blade
<label class="flex items-center space-x-2">
  <input 
    x-model="highContrast" 
    @change="$el.closest('html').classList.toggle('high-contrast')"
    type="checkbox" 
    class="w-4 h-4"
  />
  <span>High Contrast Mode</span>
</label>
```

## Advanced Theme Switching

### 1. Multi-Tenant Theme System
**For white-label verticals**:
```php
// app/Helpers/ThemeHelper.php
class ThemeHelper {
    public static function getTenantTheme($tenantId) {
        return [
            'primary' => '#8b5cf6',
            'accent' => '#ec4899',
            'name' => 'luxury_estates'
        ];
    }
}
```

```blade
<!-- In layout -->
<style>
  :root {
    --color-primary-500: {{ config('tenants.' . $tenantId . '.primary') }};
    --color-accent-500: {{ config('tenants.' . $tenantId . '.accent') }};
  }
</style>
```

### 2. User-Preference Theme Storage
**Persist user's theme choice**:
```blade
<div x-data="{ 
  theme: localStorage.getItem('theme') || 'light',
  init() {
    this.$watch('theme', (value) => {
      localStorage.setItem('theme', value);
      document.documentElement.classList.toggle('dark', value === 'dark');
    });
  }
}">
  <button @click="theme = theme === 'light' ? 'dark' : 'light'">
    Toggle Theme
  </button>
</div>
```

### 3. Dynamic Color Palette API
**Allow admins to customize colors via dashboard**:
```php
// routes/api.php
Route::post('/admin/theme/update', function (Request $request) {
    $colors = $request->validate([
        'primary' => 'string',
        'accent' => 'string',
    ]);
    
    // Save to cache or database
    Cache::put('theme.colors', $colors, forever: true);
    
    return response()->json(['status' => 'success']);
});
```

```blade
<!-- Generate CSS from database -->
<style>
  :root {
    --color-primary-500: {{ Cache::get('theme.colors.primary', '#3b82f6') }};
    --color-accent-500: {{ Cache::get('theme.colors.accent', '#f97316') }};
  }
</style>
```

## Component-Level Theme Overrides

### Override Colors in Specific Component
```blade
<!-- Use inline CSS variables for component-specific themes -->
<div style="--color-primary-500: #8b5cf6; --color-text: #000;">
  <button class="bg-[var(--color-primary-500)] text-[var(--color-text)]">
    Custom Button
  </button>
</div>
```

### Create Themed Component Variants
```blade
{{-- resources/views/components/button.blade.php --}}
@props(['variant' => 'primary', 'theme' => 'default'])

@php
    $variants = [
        'primary' => 'bg-[var(--color-primary-500)] text-white hover:bg-[var(--color-primary-600)]',
        'accent' => 'bg-[var(--color-accent-500)] text-white hover:bg-[var(--color-accent-600)]',
        'success' => 'bg-[var(--color-success)] text-white hover:opacity-90',
    ];
    
    $themeClass = $theme === 'dark' ? 'dark' : '';
@endphp

<button class="{{ $variants[$variant] }} px-4 py-2 rounded transition-colors {{ $themeClass }}">
  {{ $slot }}
</button>
```

## Tailwind Configuration for Custom Design Tokens

**If creating tailwind.config.js override**:
```javascript
// tailwind.config.js
export default {
  theme: {
    extend: {
      colors: {
        primary: {
          50: 'var(--color-primary-50)',
          500: 'var(--color-primary-500)',
          600: 'var(--color-primary-600)',
        },
        accent: {
          500: 'var(--color-accent-500)',
        }
      },
      spacing: {
        'var-1': 'var(--space-1)',
        'var-2': 'var(--space-2)',
      }
    }
  }
}
```

Then use in templates:
```blade
<div class="bg-primary-500 text-white p-var-4 rounded">
  Themed component
</div>
```

## Performance Notes
- CSS variables have minimal performance impact
- Dark mode toggle (adding/removing `dark` class) is instantaneous
- Avoid re-computing entire design system on every theme change
- Cache theme colors server-side to avoid runtime calculations
