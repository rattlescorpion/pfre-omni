# Performance Audit Prompt

## Purpose
Checklist and guidance for optimizing frontend bundle size, build performance, and runtime speed. Use this when analyzing performance bottlenecks, reducing initial load time, or preparing for production deployment.

## Quick Performance Checklist

- [ ] **Bundle Size**: Check production build size is < 500KB (JS + CSS combined)
- [ ] **Code Splitting**: Large pages load only required modules
- [ ] **CSS Optimization**: Unused styles purged by Tailwind PurgeCSS
- [ ] **Image Optimization**: All images compressed, lazy-loaded where appropriate
- [ ] **Caching**: HTTP caching headers set correctly
- [ ] **API Optimization**: N+1 queries eliminated, pagination used
- [ ] **Alpine.js**: Not loading Vue or unused libraries
- [ ] **Dark Mode**: CSS variables don't block rendering
- [ ] **Build Speed**: Production build completes < 30 seconds
- [ ] **Lighthouse Score**: ≥ 90 on desktop, ≥ 80 on mobile

## Build Optimization

### 1. Vite Configuration
**Location**: [vite.config.js](vite.config.js)

**Current Setup**:
```javascript
import laravel from 'laravel-vite-plugin';
import vue from '@vitejs/plugin-vue';

export default {
  plugins: [
    laravel({
      input: [
        'resources/css/app.css',
        'resources/js/app.js',
      ],
      refresh: true,
    }),
    vue(),
  ],
};
```

**Optimization Tips**:

1. **Enable Minification**:
```javascript
export default {
  build: {
    minify: 'terser',  // or 'esbuild' for faster builds
    sourcemap: false,  // Disable in production
  }
}
```

2. **Configure Code Splitting** (for large apps):
```javascript
export default {
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          // Split vendor code
          'vendor': ['axios', 'chart.js', 'flatpickr'],
          // Split Alpine separately if using both Alpine + Vue
          'alpine': ['alpinejs'],
        }
      }
    }
  }
}
```

3. **Optimize CSS**:
```javascript
import tailwindcss from 'tailwindcss';
import autoprefixer from 'autoprefixer';

export default {
  css: {
    postcss: {
      plugins: [
        tailwindcss('./tailwind.config.js'),
        autoprefixer,
      ],
    }
  }
}
```

### 3. Tailwind CSS Optimization

**File**: [resources/css/app.css](resources/css/app.css)

**Purge Unused Styles**:
```css
/* Only include directives, let Tailwind scan files */
@tailwind base;
@tailwind components;
@tailwind utilities;
```

**Tailwind Config** (auto-purges by default):
```javascript
// tailwind.config.js
export default {
  content: [
    './resources/views/**/*.blade.php',
    './resources/js/**/*.vue',
    './resources/js/**/*.js',
  ],
  theme: { /* ... */ },
  plugins: [],
}
```

**Check CSS Size**:
```bash
# Before optimization
du -sh public/build/app.css

# Should be < 50KB for typical project (Tailwind is ~50-100KB before purge)
```

## Asset Optimization

### 1. Image Optimization

**Compress Images**:
```bash
npm install --save-dev imagemin imagemin-mozjpeg imagemin-pngquant
```

**Configure in build**:
```javascript
// vite.config.js
import { defineConfig } from 'vite';
import imagemin from 'vite-plugin-imagemin';

export default defineConfig({
  plugins: [
    imagemin({
      gifsicle: { optimizationLevel: 7 },
      optipng: { optimizationLevel: 7 },
      mozjpeg: { quality: 80 },
      pngquant: {
        quality: [0.8, 0.9],
        speed: 4,
      },
    }),
  ],
});
```

**Lazy Load Images**:
```blade
<img 
  src="placeholder.svg" 
  data-src="/images/property.jpg"
  loading="lazy"
  class="w-full h-auto"
  alt="Property"
/>
```

### 2. Font Optimization

**System Fonts** (fastest, no download needed):
```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', sans-serif;
```

**If using custom fonts**, preload critical fonts:
```html
<!-- In <head> -->
<link rel="preload" href="/fonts/custom.woff2" as="font" type="font/woff2" crossorigin>
```

## JavaScript Optimization

### 1. Reduce Alpine.js Bundle

**Current**: Alpine.js 3.13 (~15KB gzipped)

**Only Load Alpine on Pages That Need It**:
```blade
{{-- Only include Alpine.js where needed --}}
@if($needsInteractivity)
  <script defer src="{{ asset('build/alpine.js') }}"></script>
@endif
```

### 2. Defer Non-Critical JavaScript

```blade
{{-- Load immediately --}}
<script src="/app.js"></script>

{{-- Load after page is interactive --}}
<script defer src="/analytics.js"></script>

{{-- Load only when needed (on user interaction) --}}
<script async src="/chat.js"></script>
```

### 3. Remove Unused Libraries

**Check package.json**:
```bash
npm ls  # Show all installed packages
npm audit  # Check for vulnerabilities
npm outdated  # Check for updates
```

**Clean Up Unused Packages**:
```bash
npm uninstall package-name
```

### 4. Code Splitting for Large Features

**Lazy Load Vue Components** (if using Vue 5):
```javascript
const LeadDashboard = defineAsyncComponent(() =>
  import('./components/LeadDashboard.vue')
);
```

**Lazy Load JavaScript Modules**:
```javascript
// Only load when needed
document.getElementById('map').addEventListener('click', async () => {
  const { initMap } = await import('./modules/map.js');
  initMap();
});
```

## Network Optimization

### 1. HTTP Caching Headers

**Set in Laravel** ([app/Http/Middleware/](app/Http/Middleware/)):
```php
public function handle($request, Closure $next)
{
    $response = $next($request);
    
    // Cache static assets for 1 year
    if (request()->hasMacroableMethod('is') && request()->is('build/*')) {
        $response->header('Cache-Control', 'public, max-age=31536000, immutable');
    }
    
    // Don't cache HTML pages
    if ($response->headers->get('Content-Type') === 'text/html') {
        $response->header('Cache-Control', 'no-cache, no-store, must-revalidate');
    }
    
    return $response;
}
```

### 2. Gzip Compression

**Enable in web server** (nginx example):
```nginx
gzip on;
gzip_types text/css text/javascript application/json application/javascript;
gzip_vary on;
gzip_comp_level 6;
```

**Verify Compression**:
```bash
curl -I -H "Accept-Encoding: gzip" https://yoursite.com
# Should see: Content-Encoding: gzip
```

### 3. CDN for Static Assets

**Serve images and CSS from CDN**:
```blade
<link rel="stylesheet" href="https://cdn.example.com/app.css">
<img src="https://cdn.example.com/images/logo.png" alt="Logo">
```

### 4. API Response Optimization

**Pagination** (reduce data per request):
```php
// In routes/api.php
Route::get('/leads', function () {
    return Lead::paginate(15);  // Not all leads at once
});
```

**Eager Loading** (prevent N+1):
```php
Route::get('/leads', function () {
    return Lead::with('owner', 'property')->paginate(15);
});
```

**Sparse Fieldsets** (only return needed fields):
```blade
<!-- Only fetch fields we need -->
<input 
  x-model="search"
  @keyup.debounce="
    fetch('/api/leads?fields=id,name,email')
      .then(r => r.json())
      .then(data => items = data)
  "
/>
```

## Runtime Performance

### 1. Alpine.js Optimization

**Avoid Large x-data Objects**:
```blade
{{-- ❌ Bad: Large object --}}
<div x-data="{ users: [], leads: [], deals: [], messages: [], ... }">

{{-- ✅ Good: Minimal state --}}
<div x-data="{ isOpen: false }">
```

**Use Debounce for Input**:
```blade
{{-- ❌ Bad: API call on every keystroke --}}
<input @input="search()" />

{{-- ✅ Good: Wait for pause before calling API --}}
<input @keyup.debounce.500="search()" />
```

**Use x-cloak to Hide Flash of Uninitialized Content**:
```blade
<style>
  [x-cloak] { display: none; }
</style>

<div x-data="..." x-cloak>
  <!-- Hidden until Alpine initializes -->
</div>
```

### 2. DataTables Optimization

```javascript
// In resources/js/app.js
$('#leads-table').DataTable({
  pageLength: 25,           // Don't show all rows
  lengthMenu: [10, 25, 50], // Limit options
  processing: true,         // Show loading
  serverSide: true,         // Fetch data on demand
  ajax: '/api/leads',       // Paginate from API
  columns: [
    { data: 'name' },
    { data: 'email' },
    { data: 'status' }
  ]
});
```

### 3. Chart.js Optimization

```javascript
// Don't render all chart data immediately
const chartCanvas = document.getElementById('chart');
if (chartCanvas) {
  // Only init when visible
  const chart = new Chart(chartCanvas, {
    type: 'line',
    data: smallDataset,  // Start with small dataset
    options: {
      maintainAspectRatio: false,
      responsive: true,
      animation: {
        duration: 0  // Disable animation in production
      }
    }
  });
}
```

### 4. Leaflet Map Optimization

```javascript
// Initialize map only when element is visible
const mapContainer = document.getElementById('map');
if (mapContainer && mapContainer.offsetParent !== null) {
  const map = L.map('map').setView([51.505, -0.09], 13);
  
  // Use tile layer with caching
  L.tileLayer('https://tile.openstreetmap.org/{z}/{x}/{y}.png', {
    maxZoom: 19,
    attribution: '© OpenStreetMap'
  }).addTo(map);
}
```

### 2. Bundle Analysis

```bash
# Using rollup-plugin-visualizer
npm run build
# Visit dist/stats.html
```

### 3. Chrome DevTools Performance

1. Open DevTools (F12)
2. Go to **Performance** tab
3. Click record, interact with page, stop recording
4. Review:
   - **FCP** (First Contentful Paint): < 1.8s
   - **LCP** (Largest Contentful Paint): < 2.5s
   - **CLS** (Cumulative Layout Shift): < 0.1
   - **TTFB** (Time to First Byte): < 600ms

### 4. Network Tab Analysis

1. Open DevTools (F12)
2. Go to **Network** tab
3. Load page and review:
   - **Requests**: Minimize number of requests
   - **Sizes**: CSS, JS, images should be compressed
   - **Waterfall**: Identify slow resources

## Performance Optimization Checklist

### Before Deployment
- [ ] Run `npm run build` and verify success
- [ ] Check bundle size: `du -sh public/build/*`
- [ ] Run Lighthouse audit (score ≥ 90)
- [ ] Test on mobile network (3G throttle)
- [ ] Verify CSS is purged (< 100KB)
- [ ] Confirm images are optimized and lazy-loaded
- [ ] Check HTTP caching headers are set
- [ ] Verify no console errors or warnings
- [ ] Test dark mode performance (no flashing)
- [ ] Load test with 100+ concurrent users

### Monitoring in Production
- [ ] Set up error tracking (Sentry, Rollbar)
- [ ] Enable performance monitoring (New Relic, DataDog)
- [ ] Monitor Lighthouse scores weekly
- [ ] Track Core Web Vitals via Google Analytics
- [ ] Alert on build size increases > 10%

## Common Performance Issues & Fixes

| Issue | Symptom | Fix |
|-------|---------|-----|
| **Unused CSS** | CSS file > 200KB | Enable Tailwind PurgeCSS |
| **Large JavaScript** | JS file > 300KB | Code split, tree-shake unused code |
| **N+1 Queries** | Page load > 2s | Use eager loading in API |
| **Unoptimized Images** | Images > 500KB each | Compress, use WebP, lazy-load |
| **Missing Caching** | Repeat visits slow | Set Cache-Control headers |
| **Render Blocking** | White screen on load | Defer non-critical JS |
| **Large API Responses** | JSON > 1MB | Paginate, use sparse fieldsets |
| **Uncompressed Assets** | Network > 2MB | Enable gzip compression |
